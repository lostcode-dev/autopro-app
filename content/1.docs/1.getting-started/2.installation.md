Este guia é voltado para **desenvolvedores** que precisam rodar o AutoPro localmente. Se você é um usuário final, basta criar sua conta em [autopro.app](/) e acessar direto pelo navegador — sem instalar nada.

## Requisitos

- **Node.js 20+**
- **pnpm 10+**
- Um projeto configurado no **Supabase**
- Credenciais do **Stripe** (para fluxos de cobrança)
- Credenciais da **Nuvem Fiscal** (para emissão de NFS-e e NF-e)

## Instalar dependências

```bash
pnpm install
```

## Configurar variáveis de ambiente

Copie o arquivo de exemplo e preencha com os valores do seu ambiente:

```bash
cp .env.example .env
```

As variáveis principais são:

| Variável | Obrigatória | Finalidade |
|---|---|---|
| `NUXT_PUBLIC_SITE_URL` | recomendada | URL pública usada em metadados e geração |
| `SUPABASE_URL` | sim | URL do projeto Supabase |
| `SUPABASE_ANON_KEY` | sim | Chave anônima para autenticação |
| `SUPABASE_SERVICE_ROLE_KEY` | sim | Chave administrativa server-side |
| `STRIPE_SECRET_KEY` | sim (billing) | Chave secreta do Stripe |
| `STRIPE_WEBHOOK_SECRET` | sim (billing) | Verificação de assinaturas do webhook |
| `STRIPE_ALLOWED_PRICE_IDS` | sim (billing) | IDs de preços aceitos (separados por vírgula) |
| `NUVEM_FISCAL_CLIENT_ID` | sim (fiscal) | Client ID OAuth da Nuvem Fiscal |
| `NUVEM_FISCAL_CLIENT_SECRET` | sim (fiscal) | Client Secret OAuth da Nuvem Fiscal |
| `NUVEM_FISCAL_ENVIRONMENT` | sim (fiscal) | `production` ou `sandbox` |

## Rodar em desenvolvimento

```bash
pnpm dev
```

Acesse: `http://localhost:3000`

## Verificações antes de abrir PR

```bash
pnpm lint
pnpm typecheck
```

## Onde cada coisa fica

| Diretório | O que contém |
|---|---|
| `app/pages/app/**` | Fluxo operacional da oficina |
| `app/pages/admin/**` | Fluxo administrativo global |
| `server/api/**` | Rotas server-side (Nitro) |
| `supabase/migrations/**` | Migrações do banco de dados |
| `supabase/functions/**` | Webhooks externos (Stripe, Nuvem Fiscal) |

::callout{icon="i-lucide-alert-triangle"}
Integrações com banco de dados e serviços externos (Supabase, Stripe, Nuvem Fiscal) acontecem exclusivamente no servidor, via `server/api/**`. Nunca acesse o Supabase diretamente do cliente.
::
