## Base da migração

O projeto está sendo migrado por fases para reduzir risco:

1. banco de dados
2. funções server-side críticas
3. estrutura de pages
4. módulos operacionais
5. relatórios e fiscal avançado

## Decisões importantes

- o frontend novo usa **Nuxt**
- a camada server-side usa **`server/api/**`**
- o legado não deve ser copiado como arquitetura final
- o fluxo **`/app`** e o fluxo **`/admin`** são separados

## Regra prática

Antes de migrar uma page, confirme:

- rota de destino
- APIs necessárias
- permissões necessárias
- se ela pertence a `/app`, `/admin` ou rota pública
